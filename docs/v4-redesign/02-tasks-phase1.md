# 02. 1차 페이즈 Task 분해

> 출처: `01-ia-gap-map.md` 1차 페이즈 행
> 무게중심: 매출/전환 (Sub-goal S1~S5)
> Task 단위: 1~3 영업일 (그 이상이면 더 쪼개기)
> ClickUp 임포트는 `03-clickup-import.md`에서 정리
>
> **우선순위** (99-doc-review 정합화)
> - **P0**: 임계 경로 7 task만 — T-D02, T-D03, T-D04, T-AC01, T-BO01, T-SC01, T-DV10. PM이 매일 추적
> - **P1**: 1차 페이즈 필수 (must-open / post-open 모두 포함)
> - **P2**: 1차 페이즈 있으면 좋음 / 결정 후 진행
>
> **MVP 컷** — 1차 페이즈 안에서 실제 배포 시점 기준 (99-doc-review 섹션 5 권장)
> - **must-open**: 1차 오픈 시점에 반드시 살아 있어야 함 (배포 차단 조건)
> - **post-open**: 1차 오픈 후 보강 (오픈 다음 1~4주 안에 활성화)
> - **phase-2**: 2차 페이즈로 이관 (1차 오픈 범위 밖)
>
> **상태**: ☐ 미시작 / ◐ 진행중 / ☑ 완료
> **수용기준(AC)**: 완료의 객관적 판정 기준. 이게 모호하면 task가 안 끝남.
