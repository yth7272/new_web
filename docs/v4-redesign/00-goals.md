## 최종 목표 (North Star)

> **"디바이스 단발 구매를 넘어, 다중 진입로로 신규를 빠르게 진입시키고
> PULSE PICK·라인 슬로건으로 객단가를 올리며,
> 정품등록·멤버십으로 재구매를 일으키는 자사몰."**

이번 개편의 모든 task는 "이 North Star에 어떻게 기여하는가"로 채택/기각 판정합니다.
기여가 명확하지 않은 task는 P2 또는 2차 페이즈로 넘깁니다.

## 고정값 — Task로 다툼 금지

다음은 draft가 이미 못박은 항목입니다. 매핑표/Task 작성 시 **전제 조건**으로만 사용합니다.

- **GNB 8개**: DEVICE / BOOSTER / INNER / CALM / SCIENCE / COMMUNITY / BRAND / SUPPORT
- **메인 슬로건**: "피부에, 몸 안에서, 그리고 스트레스관리까지."
- **4단 구조 불변**: 루틴팩 → 솔로(라인 슬로건) → PICK → MY ROUTINE
- **상품명 공식**: `[상위 라인] + [하위 라인] + [제품 유형]` (예: 부스터 베어 앰플)
- **임상 표기 4종 세트**: 정량 지표 + N수 + 기간 + 측정기관/방법
- **표기 룰**: 시스템=영문 대문자, 카피=한글 감성어 병행
- **카피 톤 차별화 유지**: BOOSTER=찰떡 케미 / INNER=내부 케어 / CALM=진정·과학 / SCIENCE=증명·근거

## 페이즈 정의 (draft 기준)

페이즈는 "무엇을 만들 것인가"보다 **오픈 시점에 어떤 결과물이 준비되어 있어야 하는가**로 정의합니다. 각 결과물은 화면, 운영 문서, 데이터, 정책, 검수 완료 상태까지 포함합니다.

### 1차 페이즈 — 자사몰 오픈

1차 페이즈의 목표는 새 자사몰을 고객에게 열 수 있는 상태로 만드는 것입니다. 단순 와이어프레임이 아니라, 상품을 탐색하고 구매하고 정품등록까지 이어지는 핵심 흐름이 실제 운영 가능한 수준이어야 합니다.

| 결과물 패키지 | 오픈 시 반드시 있어야 하는 결과물 | 완료 판단 기준 |
|---|---|---|
| **IA / 공통 네비게이션** | GNB 8개 메뉴 구조, 데스크톱 메가메뉴, 모바일 햄버거 메뉴, 푸터 링크 재배치, 검색/장바구니/회원 진입 위치 | DEVICE / BOOSTER / INNER / CALM / SCIENCE / COMMUNITY / BRAND / SUPPORT가 모두 노출되고, 준비중 메뉴는 클릭 처리와 문구가 정해져 있음 |
| **메인 페이지** | Hero 카피·비주얼, 추천 빠른 진입 바, 4라인 카드, SCIENCE 티저, ROUTINE LAB 인기 루틴, PHILOSOPHY 티저 | 신규 방문자가 3클릭 이내에 대표 PDP, PULSE PICK, 정품등록, SUPPORT 중 하나로 진입 가능 |
| **공식몰 자산 인벤토리** | 기존 상품/SKU, 가격, 구성품, 이미지, 리뷰, FAQ, 공지, 뉴스, 정책, 매장 정보 목록 | 신규 IA에 이관할 자산과 새로 제작할 자산이 항목별로 구분되어 있음 |
| **DEVICE 허브** | 듀얼테트라/프로/헤일로 진열, 모델 비교표, 액세서리/정품 부품 카테고리, 사용법 튜토리얼 진입, 정품등록 CTA | 고객이 모델 차이, 사용법, 구매 후 해야 할 행동을 한 페이지에서 이해 가능 |
| **디바이스 PDP 3종** | 듀얼테트라, 듀얼테트라 프로, 듀얼테트라 헤일로 PDP에 SCIENCE 6요소 표준 모듈 적용 | 헤드라인, 메커니즘, 성분/구성, 임상, 인증, CTA가 모두 있고 임상 4종 세트 표기가 빠지지 않음 |
| **정품 인증번호 등록** | 시리얼 입력 폼, 구매처/구매일 입력, 회원 매핑, 개인정보 동의, 중복 등록 차단, 리워드 정책, A/S 연결 워크플로우 | 등록 완료 후 CRM 이벤트와 A/S 대응에 필요한 정보가 남고, 비회원 시작 후 회원가입 연결 방식이 정의됨 |
| **BOOSTER 카테고리** | 부스터 라인명·슬로건 6개, 상표권 검색 결과, 라인별 진입 페이지, 대표 SKU 매핑, 루틴팩 구조 | 데일리 글로우/펌업/라이트/베어/클리어/컴포트가 상품명 공식과 PDP 카피에 일관 적용됨 |
| **PULSE PICK** | 고민별 추천 조합표, 추천 이유, 사용 순서, 개별 담기/세트 담기 CTA, 운영 룰 | 탄력/미백/진정/모공/수분/슬립 등 주요 고민별로 디바이스와 부스터 조합이 정의됨 |
| **다중 진입로** | 라인별, 고민별, 유형별, 선물하기 진입 UI와 필터/탭 와이어프레임 | BOOSTER 중심으로 먼저 동작하고, INNER/CALM은 2차 확장 가능한 구조로 남아 있음 |
| **선물하기** | 시즌, 가격대, 받는 사람별 큐레이션 룰, 포장/카드/익명/영수증 분리 옵션 검토 | 상시 카테고리인지 시즌 캠페인인지 결정되어 있고 주문 플로우 영향이 정리됨 |
| **SCIENCE 허브** | SCIENCE 진입 페이지, DEVICE SCIENCE, BOOSTER SCIENCE 초안, 임상 데이터 라이브러리, R&D/인증/파트너십 목록 | 모든 근거 문구에 출처, N수, 기간, 측정기관/방법이 붙을 수 있는 구조 |
| **COMMUNITY 1차 범위** | ROUTINE LAB MVP, 제품 태그, 좋아요, PDP 연결, 리뷰 통합 설계, 리뷰 이관/검수 기준 | 루틴/리뷰 콘텐츠가 구매 연결에는 기여하지만 의료적 효능 주장 모니터링 룰 안에 있음 |
| **BRAND / PHILOSOPHY** | PHILOSOPHY 7섹션, 회사 소개/뉴스/매거진 아카이브 재배치, 비앙글로 노출 여부 정리 | 브랜드 설명이 제품 구매와 SCIENCE 신뢰로 이어지는 CTA를 가짐 |
| **SUPPORT** | FAQ, 1:1 문의, 배송/반품, A/S 정책, 스킨케어/디바이스 사용법 교차 링크 | 정품등록, A/S, 문의가 분리되지 않고 같은 고객지원 흐름으로 연결됨 |
| **회원 / 마이페이지** | 로그인/회원가입 진입, 정품등록 hook, 구매내역, 정품등록 현황, 문의, 리뷰 섹션 | 정품등록과 CRM 이벤트를 회원 ID 기준으로 추적 가능 |
| **검수 / 배포 기준** | 임상·효능 카피 법무 검수, 리뷰 표현 모니터링 룰, URL 리다이렉트 매핑, UX 클릭패스 점검, KPI 측정 가능성 확인 | 배포 판정서에서 S1~S5 KPI 측정 가능 여부가 확인됨 |

1차 페이즈에서 하지 않는 것도 명확히 둡니다. 멤버십 등급제의 정식 자동 상향, 라이브 커머스 정기 운영, 앰버서더/체험단 정식 페이지, INNER/CALM 정식 상품 론칭, MY ROUTINE 셀프 큐레이션은 1차 오픈 범위가 아닙니다. 다만 2차에 붙일 수 있도록 메뉴, 데이터, CTA 자리만 막히지 않게 설계합니다.

### 2차 페이즈 — 운영 안정화 후

2차 페이즈의 목표는 1차에서 만든 구매/정품등록/리뷰 기반 위에 재구매와 커뮤니티 운영을 얹는 것입니다.

| 결과물 패키지 | 2차에서 만들어야 하는 결과물 | 착수 조건 |
|---|---|---|
| **멤버십 등급제** | 뉴비/펄서/펄스프로/펄스VIP 등급 정책, 디바이스 구매자 자동 상향, 등급별 혜택 페이지, 약관 | 정품등록 데이터와 회원 ID 매핑이 안정적으로 쌓인 뒤 |
| **라이브 커머스** | 라이브 예고/다시보기 페이지, 상품 연결, 표시광고법 가이드, 진행자 스크립트, 방송 후 리뷰/쿠폰 운영 룰 | 1차 상품 진열과 PULSE PICK 운영 데이터 확보 뒤 |
| **앰버서더 / 셀러브리티 / 체험단** | 협업 인물 페이지, 계약 만료 시 콘텐츠 내림 구조, 체험단 모집/선정/후기 운영 페이지 | 계약·초상권·후기 표현 정책 확정 뒤 |
| **EWG / 지속가능성** | 글로벌 대응용 원료/패키지/지속가능성 페이지, 인증·근거 자료 | 해외 진출 또는 외부 인증 활용 계획 확정 뒤 |
| **INNER 정식 론칭** | 이너 루틴팩, 이너 솔로 5개 라인, INNER PICK, INNER SCIENCE, 흡수/작용 원리 콘텐츠 | 제품, 임상, 표시 가능 표현이 준비된 뒤 |
| **CALM 정식 론칭** | 캄 루틴팩, 캄 솔로 라인, CALM PICK, CALM SCIENCE, 향/수면/무드 표현 가이드 | 첫 SKU와 효능 표현 수위가 법무 검토를 통과한 뒤 |
| **MY ROUTINE** | 사용자가 직접 루틴을 저장·공유하는 셀프 큐레이션, 제품 태그, 장바구니 연결 | ROUTINE LAB MVP 운영 후 사용자 행동 데이터 확인 뒤 |
| **AI Dr. J** | 안내 페이지, 공식 AI 표시, 면책 문구, 추천 룰, 상담/진단 오인 방지 UX | 법무와 운영팀이 AI 답변 범위와 모니터링 체계를 승인한 뒤 |

> **주의**: INNER/CALM은 1차 페이즈에서는 "준비중 노출 또는 라인 슬로건 사전 정의"만 진행합니다.
> 실제 제품/임상 준비 전 정식 카테고리로 노출하면 S4(신뢰→전환)에 역행합니다.

## 부서별 협업 필수 항목 (draft에서 추출)

- **R&D**: 임상 데이터·작용 메커니즘 1페이지 요약 / 인증·특허 / 논문 이력
- **개발**: 정품 인증번호 등록 시스템 / 멤버십 등급 로직 / 라이브 송출 인프라
- **CS**: 정품 등록 → A/S 자동 연결 / 멤버십 혜택 안내
- **법무**: 임상·효능 표기 규제 / 멤버십 약관 / 라이브 커머스 표시광고법
- **MD**: SKU 마스터, 라인별 대표 SKU, PULSE PICK 조합
- **콘텐츠**: PDP 표준 카피, 임상 표현 검수
- **PR**: 라이브·체험단·뉴스·매거진 아카이브
- **CRM**: 정품등록 리워드, 멤버십 등급 혜택, 추천 리워드

## 운영 주의사항 (의사결정 시 반복 확인)

- 라인 슬로건 변경은 카피·패키지·광고·SNS 전부 영향 → **확정 전 상표권 검색 필수**
- 정품 인증번호 등록은 개인정보 수집 동반 → 동의 절차·보안 인프라 사전 설계
- 멤버십 등급 혜택은 축소 어려움 → 도입 전 수익성 시뮬레이션
- 라이브 커머스 표시광고법 → 실시간 효능 주장 사전 가이드라인
- 앰버서더 계약 만료/리스크 → 콘텐츠 즉시 내릴 수 있는 운영 구조
- 커뮤니티 콘텐츠(루틴랩·리뷰) 의학적 효능 주장 모니터링 체계
