CuraPulse 자사몰 IA Master Board v0.1 Draft

v4 마케팅 가이드와 260513 회의 내용을 한 장에 합친 의사결정 보드입니다. 목적은 “전체 구조를 보는 것”이 아니라, 각 IA 노드를 진행 / 수정 / 보류 / 숨김 / 삭제로 판정하고 바로 ClickUp 태스크로 바꾸는 것입니다.
회의용 초안단일 HTML구현 가능 / 어려움 / 숨김 판정기준: Marketing Brief v4 + 음성회의

1. Strategy Bridge — ON / IN / MIND ↔ 실제 GNB

회의에서 가장 혼란스러웠던 지점: ON/IN/MIND는 마케팅 대전제, DEVICE/BOOSTER/INNER/CALM은 실제 자사몰 클릭 메뉴.

2. 회의 컨트롤 — 검색 / 필터 / 상태값

3. GNB IA Tree — 8개 메뉴 전체 그림

각 카드에는 해당 GNB의 정체성, 핵심 하위 노드, 필요한 결과물이 들어갑니다.

4. Node Cards — 노드별 결과물 / 완료기준 / 결정사항

상태와 메모는 브라우저 localStorage에 저장됩니다. 회의 중 판정 후 CSV로 추출하세요.

5. Multi-Entry Matrix — 라인별 / 고민별 / 유형별 / 선물하기

단순 카테고리가 아니라 상품 태그, 필터, 진열 룰, 카피가 동시에 필요합니다.

6. Main Page Flow — 고객이 첫 화면에서 이해해야 하는 흐름

7. System Dependency Map — 개발팀/외주가 먼저 봐야 하는 영역

IA 노드가 실제 시스템 기능으로 변하는 지점입니다. 여기서 난이도와 범위를 판정합니다.

8. Decision Backlog — 미결 의사결정

미결 사항을 확정처럼 쓰지 않기 위해 별도 표로 분리했습니다.

9. ClickUp 변환 규칙 — 보드에서 태스크로

생성 규칙

GO는 실행 태스크, MODIFY는 수정기획 태스크, DEV CHECK는 개발 검증 태스크, POLICY CHECK는 법무/정책 검토 태스크, SKU CHECK는 상품/운영 검토 태스크로 변환합니다. HIDE/LATER는 백로그, DROP은 제외합니다.

ClickUp 필드 초안

List: GNB 또는 담당영역
Task: 노드별 산출물명
Deliverable: 실제 결과물
Acceptance Criteria: 완료 판정 기준
Dependency: 선행조건
Risk: DEV/POLICY/SKU/R&D/CONTENT
Memo: 회의 중 입력한 메모

10. 기준 자료 / 사용 주의

이 보드는 원문 문서를 대체하지 않습니다. v4 가이드와 음성회의 결정사항을 한 화면에서 토론하기 위한 시각적 인덱스입니다. 미결 사항은 반드시 Decision Backlog에 남기고, 법무/개인정보/표시광고/임상표현 검토가 필요한 항목은 확정 태스크로 바로 넘기지 마세요. 특히 AI 댓글 자동화와 추천 리워드는 정책 검토 후 구현 범위를 결정해야 합니다.
